Sizing

  • use % to size boxes
  • use em to size texts, you can change font size of the entire website by just changing a single base font size

Box model

  • inline elements can only have margins and padding applied to the left and right, but not top and bottom
  • you can change how an element is displayed by setting display: {inline, block, none, ...}
  • trick: you can set left,right margin to auto to make a box appear in the middle
  • some methods to end the float
    1. in the element beneath the floating ones, add attribute clear: {left, right, both}
    2. in the wrapper of the floating elements, add overflow:hidden. (Bad for dropdown menus)
    3. (the rather complicated) ::after method
  • It is common practice to use ul to contain sets of links for things like navigation, menus, etc
  • Margin is out of control of box-sizing: border-box. So if you want to display 4 boxes in a row by setting width: 25% and some margin, you'll end up with 2 lines.

Flexbox

  • flex container (display: flex;) & flex item (flex: {some_property})
  • main axis (horizontal) vs. cross axis (vertical), fliped if flex-direction: column
  • elements are placed along the main axis
  • flex item properties
    • flex-grow
    • flex-direction
    • justify-content: {flex-start, flex-end, center, space-between, space-around, space-evenly}; determines how item is placed along the main axis
    • align-items: {flex-start, flex-end, center, baseline}; determines how item is placed along the cross-axis
    • flex-wrap
    • align-content

Positioning

  • static: default positiong, does not have access to top, right, bottom, left property
  • absolute: ignores content flow, if its parent does not have position set, the values set (such as top, left, etc) will be against the whole page. If the parent does have position property set (say, relative), the values set will be against its parent.
  • relative: respects content flow, makes the element a separate canvas, everything inside it that is absolutely positioned takes its place based on the dimensions of the parent
  • fixed: does not move when scrolling, biggest difference with absolute; always positioned relative to the html element
  • sticky: will scroll with other elements until it hits the set position
